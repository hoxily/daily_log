本来想趁着中午午休时间，去刷一下腾讯的全民飞机大战每日任务。一想起沈永超这破事儿就没了心情。另外，很早之前就有打算分析一下全民飞机大战使玩家沉迷的套路。干脆不打飞机了，来写写全民飞机大战的坑点。

很早很早之前，全民飞机大战只有少量飞机，飞机也不是非常变态。到了现在，不仅飞机本身很变态，还多了一个更变态的超长时间无敌状态的战魂。原本用飞机打Boss需要费尽心力，现在使用新出的60级的战魂直接无脑单挑刷爆Boss。感觉它们陷入了刷数值的陷阱里面去了。包括飞机的生命值，装备，分数加成，金币加成，变态的战魂。数值高得离谱，但是并不能带来持续的快乐。玩家很快就习惯了。然后就陷入出新的更强力的装备、飞机、宠物、更高级的关卡的无穷死循环中。

每天刷它的奖励钻石的任务。包括微信公众号上的每日签到，每日一题。还有手机QQ上的手Q游戏中心的每日签到奖励。游戏内部也有每日签到。每日活跃度达到特定值就能领取奖励。一般为了刷钻石，都会把所有任务完成，凑够活跃度数值。特别的，里面有个双打PK的任务，通常玩家都是利用冲刺功能，冲刺到若干关卡后就不再操作，或者快速自杀以完成任务。

除去每日任务，还有每日的活动模式，里面有一些有特色的小游戏。然而，一天一换，7天一个轮回。刷久了还是会厌烦。总而言之，一个人寂寞地刷着飞机，领钻石，买Fly3贵族，买单打的冲刺，买双打的冲刺，一切以收集钻石为目的。

反正可以通过充值人民币来购买钻石，那么为什么不干脆花钱买钻石呢？你的时间非常宝贵。为什么要奢侈地浪费在刷全民飞机大战上呢？

最令人难受的一点，就是它的android与iOS无法互通。老哥换了iPhone机，又不想重头刷一遍飞机装备物品，所以也就渐渐远离了飞机大战。我也换了iPhone机，但是手头上还有退休下来的华为荣耀7 Android机，可以继续使用原来的账号角色玩。

只剩我一个人寂寞地刷飞机大战。

这个跟当初玩暗黑2私服战网很像。带领我的领路人都不玩了。只剩我自己一个人刷暗黑2；不停地加新的QQ好友，教他们怎么玩暗黑的战网，然而一段时间后还是只剩下我一个人。所以我还加了一个战队，加入了战队的QQ群等等。

越是寂寞无聊空虚的情况下，玩这种类型的游戏，只会陷入更深的寂寞空虚无聊之中。

当然，后面出现了球球大作战这种实时语音交流、实时对战、小团队合作的游戏。这种强社交的游戏，是一大进步。但是，我还是带不动身边的人去一起玩一款游戏。就比如沈永超，完全带不动。也可能是我老了，身边认识的人已经过了玩游戏的年龄。另外杨介笑也带不动。杨文鹏也带不动。

MegoGames.JinYun.Alert命名空间，里面有两个类。一个是AlertView，用于控制UI界面。另一个是AlertModule，用于逻辑控制。Alert模块提供了通用的弹窗提醒功能。该弹窗需要用户确认才会关闭。另外，当前的Alert实现，看起来像我们之前的做法一样，也是同一时刻只能弹出一个弹窗来的。也就是如果因为网络消息或者别的什么原而连续创建两个弹窗的话，前一个弹窗是被后一个弹窗替换掉了。

由于使用了模块化的结构。所以别的模块在使用Alert模块来弹出窗口时，不是直接调用Alert类的方法，而是借助框架的发消息功能，来唤起通用弹窗，在EventMessage对象里塞入需要的参数。

Alert模块主要是在 public void OnWakeUp(EventMessage evt) 方法里处理其他模块的唤起消息。

EventMessage.stringValue 字段是弹框的主要提示文本；
其他参数是通过 EventMessage.Add(string key, object value) 方法添加的。可用的参数以及其含义如下：

参数名         参数类型  参数的含义                        是否可以不提供
title          string   标题                             是
normal.label   string   黄色按钮上显示的文本               是
normal.action  Action   黄色按钮被点击时的响应处理         是
green.label    string   绿色按钮上显示的文本               是
green.action   Action   绿色按钮被点击时的响应处理         是
red.label      string   红色按钮上显示的文本               是
red.action     Action   红色按钮被点击时的响应处理         是

可以按照颜色等需求自由组合。最多提供3个响应按钮。

MegoGames.JinYun.Toast命名空间下，有三个类。ToastModule，ToastView以及ToastView的内部类ToastView.ItemView。参考Android中的Toast，它是一种向用户显示信息的机制，与对话框不同，Toast没有焦点。Toast的显示时间有限，超过一定时间后会自动消失。

由于只提供了要显示的Toast消息一种参数，连延迟关闭的时间都没有。所以调起该模块的代码只需要一行，例子如下：

Post(CreateModuleWakeUpEvent("Toast", "加入失败，请重新登录游戏后重试！"));
