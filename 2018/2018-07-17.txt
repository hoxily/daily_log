Alert模块使用注意事项：
1. 注意不要写错参数名；
2. 创建好EventMessage并赋好参数之后，一定要记得使用Post方法将消息发出去；
推荐的写法，使用花括号包裹填充参数语句，创建一个代码片段来复用更好：
var alert = CreateModuleWakeUpEvent("Alert");
{
    // ...  在这里填充Alert参数，
    // 使用 Post(CreateModuleSleepEvent("Alert")); 语句来关闭弹框。
}
Post(alert);
3. Alert模块当前的实现是一个模态对话框，并且同一时刻仅存在一个对话框。如果用户没有响应前一个对话框，而又弹出来一个新的Alert对话框的话，会造成前一个对话框被替换掉；

针对第3点，他们应该有注意到这一点。所以像是网络错误这种非线性业务逻辑的弹框提示，估计是另外专门做的一个UI界面。
