2019年07月22日, 星期一, GPS(30.3107, 120.0907)

可以试试跟踪钉钉上的总群的群成员变动。每天记录数据。成员有多少，增减了多少，哪些人进来，哪些人离开。那些人都是什么部门，什么职位。

还真是有点麻烦。暂时不能自动化。借助钉钉网页版，暂时只能扒下来uid与userName两个字段。部门与职位大概也不是很关心吧。仅仅看看人头数变动应该够用了。每天更新两次应该够用。比如09:00一次，21:00一次。

目前已经有三处地方出现了要求可以拖动并拖出列表，在整个屏幕范围内移动的列表项。如果要重构的话，只能是实现一个继承自IUListItemView的abstract基类，用于实现拖动的事件监听。具体界面的UI与逻辑得交给子类来实现。另外由于加上了拖动，导致原先的Button Click需要改为IPointerClickHandler，否则会发生冲突。

删除dev分支再建最新的dev感觉怪怪的。已有的clone库都得删除一遍dev分支并从远程下载最新的dev。这样子的话，分支就是超干净的状态了。但是这样的话，容易造成有人在dev分支上的work丢失。

使用 git push --delete origin dev 就能把远程服务器上的dev分支也删除掉了。
